## 项目定位

MAX API 是面向合法授权场景的 AI 模型治理、AgentOps、AI API 管理与网关基础设施，默认面向自用、团队内部、研究机构、企业私有化部署和合规运营环境。核心能力包括：

- 企业内部统一鉴权与多模型路由
- 模型、渠道、分组、令牌和价格规则治理
- Agent 工作负载的访问控制、调用观测和成本归因
- 组织内用量统计、权限控制、成本分摊与账务管理
- 私有化部署、日志留存、审计和开发测试

## 合法授权要求

用户在使用 MAX API 时，必须满足以下要求：

- 合法取得上游 API Key、模型服务、支付服务和其他第三方服务授权
- 遵守上游服务条款、平台规则和内容政策
- 遵守所在地法律法规、监管要求和 AI 治理框架
- 对用户协议、隐私政策、实名、日志留存、税务、支付和消费者保护等事项自行承担责任

## 使用边界

使用本项目时，应确保相关场景合法授权、合规可审计，并符合以下要求：

- 不将来源不明的上游账号、密钥、额度或中转服务接入生产环境
- 不绕过上游服务条款、地域限制、内容安全或费用规则
- 不使用本项目生成、传播或协助生成违法违规、侵害权益、危害公共安全或网络安全的内容
- 日志审计、内容留存等敏感能力应仅在具备合法依据、明确告知、权限隔离和数据安全措施的场景下启用

## 公众服务义务

如用户面向不特定公众提供生成式 AI 服务，应自行完成所在地法律法规要求的合规义务，包括但不限于：

- 备案、许可或资质要求
- 内容安全审核与滥用治理机制
- 身份管理、用户协议、隐私政策与日志留存
- 税务、支付、发票、消费者保护和支付风控
- 上游授权、服务条款和模型内容政策遵循

<Callout type="warn">
  MAX API 提供模型与 Agent 工作负载的网关治理层，不提供上游模型账号、API Key、基础模型训练能力或模型服务本身。对外提供服务所涉及的合规义务，由部署方自行承担。
</Callout>

## 密钥与渠道要求

- 上游渠道必须是部署方合法拥有或已获授权的账号、API Key、模型服务或企业合约
- 多密钥管理、负载均衡、故障切换等功能仅用于高可用、组织治理和企业多账号管理
- 使用模型映射、路径覆盖、参数覆盖、通用视频任务协议等能力时，应确保最终请求符合上游接口规则和授权范围

## 支付与账务管理

支付、充值、兑换码、邀请、订阅、余额和计费功能仅用于合法授权场景下的内部结算、企业客户账务或合规服务收费。部署方应确保相关服务、收费、推广、税务、发票和上游授权均符合所在地法律法规、上游服务条款和平台规则。

## 社区治理

本项目社区讨论应围绕合法授权、合规部署和正当工程问题展开。Issue、PR、Discussion、教程和社区讨论应符合项目合规要求和社区规则，维护者将按规则处理不符合要求的内容。
