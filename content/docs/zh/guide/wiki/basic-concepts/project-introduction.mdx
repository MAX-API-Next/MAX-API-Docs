## 概述

MAX API 是由来自科研机构和高校的 AGI 爱好者组织发起、维护和长期运营的开源项目，面向开发者、研究者、团队和组织提供统一的 AI 模型接入、鉴权、路由、计费、观测和治理层。

它不只是请求转发器，而是面向 AI-ready 应用和 Agent 工作负载的可运营网关。MAX API 将模型供应商、协议格式、渠道能力、令牌权限、成本核算、日志审计、限流、缓存和多模态任务纳入统一治理边界，帮助部署方更稳定、更可控地运行 AI 应用。

<Callout type="info">
  MAX API 不提供上游模型账号、API Key、基础模型训练能力或模型服务本身。部署方需要自行取得合法授权的上游服务，并承担相应合规义务。
</Callout>

## 核心定位

- **模型治理平面**：统一管理模型入口、渠道、供应商、协议格式、模型映射、价格规则和多模态接口。
- **AgentOps 控制平面**：为 Agent、工作流和工具调用提供独立令牌、模型访问控制、调用日志、成本归因、异常定位和运营分析。
- **协议与供应商适配层**：兼容 OpenAI Compatible、Responses、Claude Messages、Gemini、Realtime、嵌入、重排序、图像、音频和视频任务等接口。
- **渠道配置治理**：在渠道新建和编辑中展示能力矩阵、配置校验、模型发现、路径覆盖和任务协议模板，降低误配置风险。
- **成本与可靠性治理**：支持表达式计费、分阶段计费 JSON、任务 rate-card、倍率计费、预扣费、失败退款、渠道加权随机、失败重试和模型限流。
- **性能与可扩展性治理**：支持 Redis / 内存缓存、流式超时、大请求体限制、上游连接池调优、磁盘缓存、Pyroscope 性能采样和优雅关闭。

## 适用场景

- 组织内部 AI 模型治理平台
- Agent 应用运行与治理底座
- 多供应商与国产模型适配中心
- 文本、图像、视频、音频、嵌入、重排序和实时对话的多模态任务治理平台
- 模型成本与 Agent 成本核算平台
- 私有化与合规运营环境
