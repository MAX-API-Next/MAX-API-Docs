title: 操练场

这里可以快速进行 AI 聊天，方便测试模型、渠道和请求参数。

![操练场](/assets/guide/feature-guide/playground.png)

## 选择模型和分组

在输入框上方选择模型和可用分组。选择 `auto` 或 `auto:*` 分组时，操练场会加载当前用户可用的模型集合；实际可用性仍取决于管理员配置的渠道、分组和权限。

## 模型参数

点击输入框工具栏中的参数按钮可打开参数设置。桌面端显示为弹出面板，移动端显示为底部面板。每个参数都需要单独启用，**只有已启用的参数才会随聊天补全请求发送**。设置会保存在当前浏览器中，便于下次继续测试。

| 参数 | 范围 | 用途 |
| --- | --- | --- |
| 温度（Temperature） | `0` - `2`，步进 `0.1` | 控制随机性和创造性。较低的值通常更稳定，较高的值通常更发散。 |
| Top P | `0` - `1`，步进 `0.05` | 将候选 Token 限制在指定的累计概率范围内。 |
| 频率惩罚（Frequency Penalty） | `-2` - `2`，步进 `0.1` | 降低重复用词的倾向。 |
| 存在惩罚（Presence Penalty） | `-2` - `2`，步进 `0.1` | 鼓励模型引入新的主题。 |
| 最大 Token 数（Max Tokens） | `1` - `200000`，整数 | 限制回复长度。 |
| 随机种子（Seed） | `0` - `2147483647`，整数，可留空 | 在支持的模型上提高相同请求的可重复性。留空时不会发送 `seed`。 |

数值会按控件范围和步进值规范化。清空必填数值会回落到该参数的最小值；随机种子可清空为未设置状态。

> 模型和上游服务对参数的支持程度不同。即使参数已启用，不支持该字段的上游仍可能返回参数校验错误；遇到这类情况请关闭相应参数或选择兼容模型。
