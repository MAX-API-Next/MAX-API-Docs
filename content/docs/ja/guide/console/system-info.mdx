---
title: システム情報とインスタンス
---

システム情報ページは Root 管理者向けで、マルチノード環境のインスタンス状態、リソース使用量、システムタスクを確認できます。プロフィールメニューから **システム情報** を開きます。

## インスタンスの状態

インスタンス一覧には、ノード名、ロール、バージョン、起動時刻、最終ハートビート時刻、リソース状態が表示されます。インスタンスは既定で 30 秒ごとに状態を送信します。最終ハートビートが現在時刻から 90 秒より前のレコードは失効（`stale`）として扱われます。

ローリングアップデートや非同期タスクの調査中にも同じインスタンスを識別できるよう、各ノードには安定して一意な `NODE_NAME` を設定してください。

## 失効したインスタンスの削除

インスタンスレコードを削除できるのは Root 管理者だけです。

1. **システム情報 -> インスタンス** で削除する失効レコードを確認します。
2. 単一レコードの削除では、オンライン中または失効しきい値内のインスタンスはバックエンドにより拒否されます。
3. **すべての失効を削除** を使うと、すべての失効レコードを一度に削除できます。確認後に一覧は自動更新されます。

一括削除の対象は、`last_seen_at` が現在時刻から失効しきい値を引いた時刻より**前**のレコードだけです。オンライン中のインスタンスと、しきい値ちょうどのレコードは削除されません。処理中は重複操作を防ぐため、更新と単一インスタンスの削除が一時的に無効になります。

## 管理 API

Root 専用エンドポイントは `DELETE /api/system-info/stale-instances` です。成功時の `data.deleted_count` は実際に削除されたレコード数です。この操作は既存の `system_instances` データを再利用し、データベース移行は不要です。

> インスタンスレコードを削除しても、プロセスの停止、キーの無効化、チャネル設定の変更は行われません。まだ稼働している可能性があるノードを削除する前に、ネットワーク、時刻同期、ハートビート送信を確認してください。
