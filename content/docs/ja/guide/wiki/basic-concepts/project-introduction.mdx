## 概要

MAX API は、研究機関や大学の AGI 愛好者によって立ち上げられ、保守・運営されているオープンソースプロジェクトです。開発者、研究者、チーム、組織に向けて、AI モデルへのアクセス、認証、ルーティング、課金、可観測性、ガバナンスを統一するレイヤーを提供します。

単なるリクエスト転送器ではありません。MAX API は、AI-ready アプリケーションや Agent ワークロードを運用するためのゲートウェイです。モデルプロバイダー、プロトコル形式、チャネル能力、トークン権限、コスト計算、ログ監査、レート制限、キャッシュ、マルチモーダルタスクを一つのガバナンス境界にまとめ、より安定して制御可能な AI アプリケーション運用を支援します。

<Callout type="info">
  MAX API は、アップストリームのモデルアカウント、API Key、基盤モデルの学習機能、またはモデルサービスそのものを提供しません。デプロイする側が、合法的に認可されたアップストリームサービスを取得し、対応するコンプライアンス義務を負います。
</Callout>

## コアポジション

- **モデルガバナンスプレーン**：モデル入口、チャネル、プロバイダー、プロトコル形式、モデルマッピング、価格ルール、マルチモーダルインターフェースを一元管理します。
- **AgentOps コントロールプレーン**：Agent、ワークフロー、ツール呼び出しに対して、独立したトークン、モデルアクセス制御、呼び出しログ、コスト帰属、障害診断、運用分析を提供します。
- **プロトコルとプロバイダーの適配レイヤー**：OpenAI Compatible、Responses、Claude Messages、Gemini、Realtime、埋め込み、リランキング、画像、音声、動画タスクなどの API に対応します。
- **チャネル設定ガバナンス**：チャネル作成・編集時に、能力マトリクス、設定検証、モデル発見、パス上書き、タスクプロトコルテンプレートを表示し、設定ミスを減らします。
- **コストと信頼性のガバナンス**：式ベース課金、段階課金 JSON、タスク rate-card、倍率課金、事前課金、失敗時返金、チャネルの加重ランダム、失敗時リトライ、モデルレート制限をサポートします。
- **性能と拡張性のガバナンス**：Redis / メモリキャッシュ、ストリーミングタイムアウト、大きなリクエストボディ制限、アップストリーム接続プール調整、ディスクキャッシュ、Pyroscope プロファイリング、グレースフルシャットダウンをサポートします。

## 想定シナリオ

- 組織内 AI モデルガバナンス基盤
- Agent アプリケーションの実行・管理基盤
- 複数プロバイダーや国内モデルの適配ハブ
- テキスト、画像、動画、音声、埋め込み、リランキング、リアルタイム会話のマルチモーダルタスク管理
- モデルコストと Agent コストの計算基盤
- プライベートデプロイおよびコンプライアンス重視の運用環境

## ライセンスと認可境界

MAX API は **GNU AGPLv3** オープンソースライセンスを採用しています。

本プロジェクトは [One API](https://github.com/songquanpeng/one-api) と [New API](https://github.com/QuantumNous/new-api) を基に開発されています。商用利用または二次開発を行う場合、MAX API の認可条件に加えて、One API の MIT ライセンスおよび New API の AGPLv3 ライセンスにも従う必要があります。MAX API プロジェクトが提供する一時的な商用認可は、アップストリームプロジェクトのオープンソースライセンス義務を置き換えたり免除したりするものではありません。

<Callout type="warn">
  本プロジェクトを変更し、ネットワークサービスとしてユーザーに提供する場合は、AGPLv3 に基づくソースコード提供などの義務を理解し遵守してください。長期商用認可や組織間協力については maxapi@max-api.ai までお問い合わせください。
</Callout>

## 二次開発とコミュニティ表示

MAX API を基に自用の二次開発プロジェクトを作る場合、プロジェクトのホームページ、フッター、About ページなどの見やすい場所で、次のいずれかの方法により出典を表示できます。

- プロジェクト URL を追加する：[MAX-API-Next/MAX-API](https://github.com/MAX-API-Next/MAX-API)
- コミュニティを表示する：[MAX-API-Next](https://github.com/MAX-API-Next)

上記いずれかの表示条件を満たし、リンクが明確に見える状態を維持している場合、プロジェクト README に記載された非永久の一時的な商用認可が自動的に有効になります。別途申請や確認待ちは不要です。この認可は、MAX API プロジェクト側が認可権限を持つ追加・変更部分にのみ適用されます。
